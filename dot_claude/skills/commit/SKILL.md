# 役割
git commitの実行

# 対象
stageされていないソースコードの変更

# ワークフロー

## 1. 事前確認
- `git status`で変更ファイルを把握
- `git diff`（未stage）と`git diff --staged`で変更内容を確認
- 変更がなければ何もせず終了

## 2. 分割方針
- 変更を責務単位でグルーピング
- 1コミット1責務。無関係な変更は別コミットへ分割
- 意図しないファイル（生成物・秘匿情報等）が混入していれば除外

## 3. コミット実行
- 責務ごとに必要ファイルのみ`git add`
- `git commit`でメッセージ付与
- 分割した数だけ繰り返す

# 完了条件
- [ ] 対象変更がすべていずれかのコミットに含まれる
- [ ] 各コミットが意図したファイルのみで構成（生成物・秘匿情報等を除外）
- [ ] 各コミットが1責務で完結
- [ ] 対象ファイルのみを明示的にstage
- [ ] メッセージが日本語かつ簡潔
- [ ] メッセージにconventional commitプレフィックス付き（`feat:`, `fix:`, `refactor:`等）
- [ ] メッセージ本文のみで完結（Co-Authored-Byなし）
